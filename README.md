# Predictive Failure Analysis Project

## Описание

Этот проект представляет собой систему для прогнозирования вероятности сбоев сервера на основе анализа данных об использовании оперативной памяти. Система состоит из трех основных частей: нейронной сети, бэкенда и фронтенда.

*   **Нейронная сеть:**  Анализирует данные об изменении оперативной памяти во времени, чтобы предсказать вероятность сбоя в следующем часе.
*   **Бэкенд (Backend):** Получает данные в формате .csv с сайта, парсит их, вычисляет изменение оперативной памяти и отправляет данные в нейронную сеть для анализа. Затем бэкенд получает результат анализа (вероятность сбоя) и отправляет его на фронтенд.
*   **Фронтенд (Frontend):**  Веб-сайт, который позволяет пользователям загружать .csv файлы с данными о работе сервера. Данные отправляются на бэкенд, где происходит их обработка. Результат анализа (вероятность сбоя) отображается на сайте в виде графика с указанием инцидентов в каждом часе.

## Функциональность

*   **Загрузка .csv файлов:**  Пользователь может загружать .csv файлы, содержащие данные о работе сервера.
*   **Парсинг данных:** Бэкенд автоматически парсит .csv файлы, извлекая необходимые атрибуты (время, наличие инцидента).
*   **Вычисление изменения оперативной памяти:** Бэкенд вычисляет изменение оперативной памяти за двухминутный интервал.
*   **Прогнозирование вероятности сбоя:** Нейронная сеть анализирует данные и предсказывает вероятность сбоя в следующем часе.
*   **Визуализация данных:** Фронтенд отображает график вероятности сбоя и инцидентов во времени.

## Технологии

*   **Нейронная сеть:**  PyTorch
*   **Бэкенд:** Python, Flask
*   **Фронтенд:** HTML, CSS, JavaScript

## Запуск проекта

1.  Клонируйте репозиторий: `git clone https://github.com/Tolex1510/Hakaton_OZRV.git`
2.  Перейдите в директорию бэкенда: `cd Hakaton_OZRV`
3.  Создайте виртуальное окружение: `python -m venv venv`
4.  Активируйте виртуальное окружение:
    *   Linux/macOS: `source venv/bin/activate`
    *   Windows: `venv\Scripts\activate`
5. Перейдите в папку Backend: `cd Backend`
6. Установите зависимости: `pip install -r requirements.txt`
7. Вернить в прошлую директорию: `cd ..`
8. Запустите сервер: `python main.py`
